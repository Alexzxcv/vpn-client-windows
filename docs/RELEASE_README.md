# VPN Client (Windows)

Десктоп-клиент VPN-сервиса. Два режима туннелирования поверх VLESS Reality:

- **Proxy** — локальный SOCKS/HTTP прокси (движок xray). Без драйверов и без прав
  администратора. Трафик идёт через прокси для приложений, поддерживающих системный
  прокси Windows (браузеры и т.п.).
- **Full tunnel (TUN)** — полный туннель на всё устройство (движок sing-box + WinTUN).
  Заворачивает весь трафик ОС. **Требует запуска от имени администратора.**

## Что в архиве

- `vpnclient.exe` — клиент (Go-ядро + встроенный React-UI в окне WebView2).
- `xray.exe` — движок proxy-режима (xray-core), запускается как субпроцесс.
- `sing-box.exe` — движок TUN-режима (полный туннель), запускается как субпроцесс.
- `wintun.dll` — драйвер виртуального сетевого интерфейса для TUN. Должен лежать рядом.

## Запуск

1. Распакуйте архив в любую папку (например `C:\Program Files\VPNClient`).
   `vpnclient.exe`, `xray.exe`, `sing-box.exe` и `wintun.dll` должны лежать рядом.
2. Запустите `vpnclient.exe`. Появится окно входа и иконка в системном трее.
3. Войдите своим email/паролем, выберите локацию, режим (Proxy / Full tunnel) и нажмите Connect.

## Полный туннель (TUN)

- Для TUN-режима запускайте `vpnclient.exe` **от имени администратора**
  (правый клик → «Запуск от имени администратора»). Без этого клиент покажет
  подсказку и не даст подключиться в режиме Full tunnel.
- В TUN-режиме системный прокси Windows не используется — маршрутизацией управляет
  виртуальный интерфейс WinTUN; туннелируется весь трафик устройства.
- Proxy-режим прав администратора не требует.

## Системный трей

- **Connect / Disconnect** — управление подключением.
- **Open** — открыть окно клиента (если оно было закрыто).
- **Quit** — полностью выйти (снимает системный прокси).

Закрытие окна сворачивает приложение в трей — оно продолжает работать.
Повторный запуск второго экземпляра блокируется.

## Требования

- Windows 10/11.
- Среда выполнения **WebView2** (предустановлена в Windows 11; для Windows 10
  ставится из «Microsoft Edge WebView2 Runtime»).

## Безопасность

- Системный прокси снимается при выходе, при падении ядра и при следующем
  старте, если остался от аварийно завершённого запуска.
- Клиент с открытым исходным кодом; секреты в дистрибутиве не хранятся —
  конфигурация тянется с бэкенда в рантайме.
