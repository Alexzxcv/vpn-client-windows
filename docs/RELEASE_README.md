# VPN Client (Windows)

Десктоп-клиент VPN-сервиса. Два режима туннелирования поверх VLESS Reality:

- **Proxy** — локальный SOCKS/HTTP прокси (движок xray). Без драйверов и без прав
  администратора. Трафик идёт через прокси для приложений, поддерживающих системный
  прокси Windows (браузеры и т.п.).
- **Full tunnel (TUN)** — полный туннель на всё устройство (движок sing-box + WinTUN).
  Заворачивает весь трафик ОС. **Требует запуска от имени администратора.**

## Что в архиве

- `vpnclient.exe` — клиент (Go-ядро + встроенный React-UI в окне WebView2).
- `xray.exe` — движок proxy-режима (xray-core), запускается как субпроцесс.
- `sing-box.exe` — движок TUN-режима (полный туннель), запускается как субпроцесс.
- `wintun.dll` — драйвер виртуального сетевого интерфейса для TUN. Должен лежать рядом.

## Запуск

1. Распакуйте архив в любую папку (например `C:\Program Files\VPNClient`).
   `vpnclient.exe`, `xray.exe`, `sing-box.exe` и `wintun.dll` должны лежать рядом.
2. Запустите `vpnclient.exe`. Появится окно входа и иконка в системном трее.
3. Войдите по **email или логину** и паролю, выберите локацию, режим
   (Proxy / Full tunnel) и нажмите Connect.

Если у вас включена **двухфакторная аутентификация (2FA)**, после ввода пароля
клиент попросит 6-значный код из приложения-аутентификатора.

Нет аккаунта? На экране входа есть кнопка **«Создать аккаунт»** — она откроет
веб-кабинет в браузере для регистрации.

## Возможности

- **Выбор локации** и режим **«Авто (лучший)»** — лучшую ноду подберёт сервис.
- **Два режима подключения:** Proxy (без прав администратора) и Full tunnel / TUN
  (весь трафик устройства, требует администратора — см. ниже).
- **Свои (кастомные) серверы.** Можно добавить чужой VLESS-сервер: вставьте
  `vless://`-ссылку или ссылку на подписку (http/https) — клиент импортирует все
  серверы из неё. Свою ссылку сервера можно скопировать, ненужный сервер — удалить.
  Трафик по своим серверам не учитывается и не ограничен подпиской.
- **«Российские сайты напрямую»** — российские сайты и адреса идут в обход VPN.
- **Список «напрямую» (split-tunnel)** — укажите домены или IP-подсети
  (`.ru`, `example.com`, `10.0.0.0/8`), которые должны ходить мимо туннеля.
- **Kill switch** — при обрыве туннеля блокирует весь трафик, чтобы он не утёк
  мимо VPN (для режима Full tunnel).
- **Автозапуск с Windows** — клиент стартует при входе в систему.
- **Авто-переподключение** — при разрыве клиент сам поднимает соединение заново.
- **Автообновление** — клиент проверяет новые версии и предлагает обновиться;
  обновление ставится только по вашему подтверждению.
- **Язык EN/RU** — переключатель языка в настройках.

Большинство переключателей (kill switch, российские сайты напрямую, список
«напрямую», автозапуск, порты) — на странице **«Настройки»**.

## Полный туннель (TUN)

- Для TUN-режима запускайте `vpnclient.exe` **от имени администратора**
  (правый клик → «Запуск от имени администратора»). Без этого клиент покажет
  подсказку и не даст подключиться в режиме Full tunnel.
- В TUN-режиме системный прокси Windows не используется — маршрутизацией управляет
  виртуальный интерфейс WinTUN; туннелируется весь трафик устройства.
- Proxy-режим прав администратора не требует.

## Системный трей

- **Connect / Disconnect** — управление подключением.
- **Open** — открыть окно клиента (если оно было закрыто).
- **Quit** — полностью выйти (снимает системный прокси).

Закрытие окна сворачивает приложение в трей — оно продолжает работать.
Повторный запуск второго экземпляра блокируется.

## Требования

- Windows 10/11.
- Среда выполнения **WebView2** (предустановлена в Windows 11; для Windows 10
  ставится из «Microsoft Edge WebView2 Runtime»).

## Безопасность

- Системный прокси и правила kill switch снимаются при выходе, при падении ядра
  и при следующем старте, если остались от аварийно завершённого запуска.
- Клиент с открытым исходным кодом; секреты в дистрибутиве не хранятся —
  конфигурация тянется с бэкенда в рантайме.
