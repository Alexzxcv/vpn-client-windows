# SAPN VPN — клиент для Windows

Десктоп-клиент VPN-сервиса SAPN для Windows. Под капотом — Go-ядро и встроенный
React-интерфейс, который открывается в нативном окне на движке **WebView2**.
Туннели строятся поверх **VLESS + Reality**.

Для нод сервиса конфигурация (VLESS Reality) тянется с бэкенда по API в рантайме —
сырая ссылка наружу не выдаётся и в дистрибутиве не хранится.

> **Публичный репозиторий.** Секретов, токенов и ключей в коде нет: всё
> чувствительное приходит с бэкенда в рантайме или задаётся через переменные
> окружения.

## Возможности

- **Вход по email или логину** — один идентификатор, либо почта, либо username.
- **Двухфакторная аутентификация (TOTP)** на входе — если у аккаунта включён 2FA,
  клиент запрашивает 6-значный код из приложения-аутентификатора.
- **Кнопка «Создать аккаунт»** — открывает веб-дашборд (регистрация / личный
  кабинет) во внешнем браузере. Адрес задаётся через env `VPNCLIENT_DASHBOARD_URL`.
- **Выбор локации** и режим **«Авто (лучший)»** — при «Авто» лучшую ноду по
  latency/health выбирает бэкенд.
- **Два режима туннелирования:**
  - **Proxy** — локальный SOCKS/HTTP-прокси на движке **xray**. Не требует прав
    администратора; задействует системный прокси Windows.
  - **Full tunnel / TUN** — полный туннель на всё устройство на движке
    **sing-box + WinTUN**. **Требует запуска от имени администратора.**
- **Пользовательские (кастомные) серверы** — вставка `vless://`-ссылки или импорт
  подписки по http(s); копирование `vless`-ссылки сервера; удаление. Подключение к
  кастомным нодам идёт напрямую по локальному конфигу, **минуя бэкенд**: без
  регистрации устройства, без `/vpn/config`, без учёта трафика и проверок подписки.
  Для подключённого кастомного сервера показывается реальный пинг (прямой TCP-замер).
- **Split-tunnel «список напрямую»** — домены и IP-CIDR (`.ru`, `example.com`,
  `10.0.0.0/8`), которые идут в обход туннеля.
- **«Российские сайты напрямую»** — geoip/geosite `ru` пускаются мимо туннеля.
- **Kill switch** — при обрыве туннеля блокирует весь исходящий трафик (firewall),
  чтобы не было утечки в обход VPN. Применяется в TUN-режиме.
- **Автозапуск с Windows** — старт при входе в систему (ключ реестра Run).
- **Авто-переподключение** — при падении движка клиент сам поднимает туннель заново
  с экспоненциальным backoff.
- **Автообновление** с GitHub Releases — фоновая проверка новых версий; установка —
  только по явному действию пользователя (клиент сам ничего не накатывает).
- **Системный трей** — Connect/Disconnect, открыть окно, выход (снимает прокси).
- **Локализация EN/RU** с переключателем языка прямо в интерфейсе.

Дополнительно: единственный экземпляр (повторный запуск блокируется),
crash-safe — системный прокси и правила kill switch снимаются при выходе, падении
ядра и при следующем старте, если остались от аварийного запуска. Подключение
переживает перезапуск (токены входа сохраняются). Идентичность устройства —
криптографическая (ключ Ed25519, приватная часть запечатана через Windows DPAPI).

## Архитектура

**Go-ядро.** Раскладка по слоям (handler → service → движки). Локальный
control-API слушает только `127.0.0.1` на свободном порту и защищён одноразовым
session-token (Bearer) плюс проверкой Origin/Host; он же отдаёт собранный React-UI
как статику. Движки **xray** (proxy) и **sing-box** (TUN) запускаются как
субпроцессы.

Основные пакеты `internal/`:

- `app` — оркестратор: машина состояний подключения, авто-реконнект,
  авто-рефреш credential, обновления.
- `control` — локальный control-HTTP-сервер для UI (`/api/*`).
- `backend` — HTTP-клиент к бэкенду (login, `/me`, locations, `/vpn/config`, usage).
- `xray` / `singbox` — генерация конфигов и управление жизненным циклом движков.
- `customserver` — хранение и парсинг пользовательских VLESS-серверов/подписок.
- `routing` — split-tunnel (домены/IP-CIDR, «российские сайты напрямую»).
- `killswitch` — firewall-блокировка egress при обрыве туннеля.
- `settings` / `tokenstore` / `deviceid` — локальное состояние
  (`%APPDATA%/sapn-vpn`), токены входа, крипто-идентичность устройства.
- `autostart` / `updater` / `sysproxy` / `elevation` / `singleinstance` —
  автозапуск, автообновление, системный прокси, проверка прав, single-instance.

**React-фронт** (`frontend/`, TypeScript + rsbuild, состояние на **MobX**). Собранная
статика отдаётся ядром и открывается в окне WebView2. Страницы: вход, подключение
(выбор локации/режима, трафик, пинг, кастомные серверы), настройки.

## Сборка и разработка

Требования:

- **Go 1.25**
- **Node 20** (для сборки UI)
- **WebView2 Runtime** (предустановлен в Windows 11; для Windows 10 ставится из
  «Microsoft Edge WebView2 Runtime»)

Рутинные команды — через `Makefile`:

- `make ui` — собрать React-интерфейс (`frontend/`).
- `make build` — собрать `bin/vpnclient.exe`.
- `make build-gui` — релизная сборка без консольного окна (stripped, с версией).
- `make run` — локальный запуск ядра.
- `make xray` / `make singbox` / `make wintun` — скачать движки и WinTUN в `bin/`.
- `make geo` — скачать гео-базы (`geoip.dat`/`geosite.dat` для xray и `.srs`
  rule-set для sing-box; используются переключателем «российские сайты напрямую»).
- `make vet` / `make test` — `go vet` / `go test ./...`.
- `make release` — собрать UI + GUI-бинарь + движки + гео и упаковать zip в `dist/`.
- `make installer` — собрать установщик `SAPN-VPN-Setup.exe` (Inno Setup, `iscc`).

Для локального запуска: `make ui && make xray && make singbox && make wintun && make run`.

### Переменные окружения

- `VPNCLIENT_API_BASE` — базовый URL бэкенда.
- `VPNCLIENT_DASHBOARD_URL` — URL веб-дашборда для кнопки «Создать аккаунт».
- `VPNCLIENT_UI_DIR` — каталог собранного UI (по умолчанию ищется рядом с exe).
- `VPNCLIENT_NO_SYSPROXY` — не трогать системный прокси (для headless-тестов).
- `VPNCLIENT_UPDATE_CHECK=0` — выключить авто-проверку обновлений;
  `VPNCLIENT_UPDATE_INTERVAL` — интервал проверки (Go-duration, напр. `6h`).
- `VPNCLIENT_DEBUG` — включить debug-логи.

### Релиз

Релиз собирается в CI по пушу тега `v*` (см. `.github/workflows/release.yml`):
собираются UI и GUI-бинарь, скачиваются `xray.exe`, `sing-box.exe`, `wintun.dll` и
гео-базы, формируются **zip** и установщик **Inno Setup** и прикладываются к
GitHub Release.
